Документация

1. Пользовательские сценарии

1.1 Первый вход пользователя

  1. Пользователь запускает бот (в Telegram или на сайте).
  1. Бот приветствует пользователя и предлагает выбрать сферу:
    • Спорт и фитнес
    • Саморазвитие и психология
    • Правильное питание и сон
  1. Пользователь выбирает сферу.
  1. Бот предлагает пройти тест (5–10 вопросов):
    • Например: "Сколько часов вы обычно спите?"
  1. По итогам теста бот генерирует список тегов и персонализирует рекомендации.
  1. Пользователь описывает свою проблему или цель в текстовом поле.
  1. Бот анализирует текст, сопоставляет с тегами и выводит подборку ресурсов:
    • Статьи, книги, приложения, товары с описанием и ссылками.

1.2 Регулярное взаимодействие

  1. Пользователь возвращается в бот.
  1. Бот предлагает выбрать:
    • Получить новые рекомендации.
    • Просмотреть предыдущие рекомендации.
    • Пройти новый тест.
  1. Пользователь взаимодействует с выбранным разделом.

1.3 Пример сценария взаимодействия

Цель пользователя: Улучшить качество сна.

Сценарий:

  1. Бот: "Привет! Как я могу вам помочь сегодня?"
  1. Пользователь: "Я плохо сплю."
  1. Бот:
    • Предлагает пройти тест: "Сколько часов вы обычно спите?"
    • Анализирует ответ и предлагает:
      • Статью: "5 советов для улучшения сна".
      • Приложение: "Calm — для медитаций перед сном".
      • Товар: "Маска для сна".

2. Структура базы данных

2.1 Таблицы

Таблица: Users (пользователи)

ПолеТипОписание
idINTEGERУникальный идентификатор пользователя.
usernameTEXTИмя пользователя или никнейм.
preferencesTEXT (JSON)Предпочтения (список тегов).
last_interactionDATETIMEДата последнего взаимодействия.

Таблица: Resources (ресурсы)

ПолеТипОписание
idINTEGERУникальный идентификатор ресурса.
titleTEXTНазвание ресурса (книга, статья, товар).
typeTEXTТип: книга, статья, приложение и т. д.
tagsTEXT (JSON)Теги для классификации.
descriptionTEXTОписание ресурса.
urlTEXTСсылка на ресурс.
priceDECIMALЦена (если применимо).

Таблица: User_Interactions (история взаимодействий)

ПолеТипОписание
idINTEGERУникальный идентификатор.
user_idINTEGERИдентификатор пользователя.
resource_idINTEGERИдентификатор ресурса.
timestampDATETIMEДата взаимодействия.

2.2 Пример связей

  • Один пользователь может взаимодействовать с несколькими ресурсами (связь User_Interactions).
  • Каждый ресурс может быть связан с несколькими тегами.

3. Технические детали

3.1 Стек технологий

  • Backend: Python (Flask/FastAPI), PostgreSQL для хранения данных.
  • Frontend: HTML/CSS/JavaScript (React для сайта).
  • Бот: Telegraf.js (Node.js) или Python-бот через aiogram.
  • Хостинг: AWS или Heroku.

3.2 API-интеграции

  1. OpenAI GPT API:
    • Используется для анализа текста пользователя и генерации тегов.
  1. Spoonacular API (для питания):
    • Рекомендации рецептов и информации о продуктах.
  1. GoodReads API (для саморазвития):
    • Информация о книгах и рейтингах.

3.3 Логика работы бота

  1. Анализ текста:
    • Пользователь пишет запрос.
    • GPT API возвращает ключевые слова (теги).
  1. Поиск ресурсов:
    • Теги используются для фильтрации таблицы Resources.
    • Возвращаются релевантные ресурсы.
  1. Ответ пользователю:
    • Формируется сообщение с описанием и ссылками.

3.4 Хранение данных

  • Данные о пользователях:
    • Анонимизированные идентификаторы.
    • Хранятся только предпочтения и история взаимодействий.
  • Ресурсы:
    • Локальная база данных, обновляемая из внешних API.

3.5 Риски и ограничения

  1. Низкая скорость ответа:
    • Решение: Кеширование популярных запросов.
  1. Ошибки классификации тегов:
    • Решение: Улучшение предобучения GPT API.
